===========================================
  "从服务器返回了一个参照" 弹窗解决方法
===========================================

【原因】
UIAccess 程序有两个硬性要求，缺一不可：
  1. EXE 必须经过代码签名（不能是未签名的）
  2. 签名所用的证书必须在"本地计算机\受信任的根证书颁发机构"中

只安装证书但 EXE 没有签名 → 仍然弹窗！
只签名但证书不受信任 → 仍然弹窗！

【正确流程（必须按顺序）】
  1. scripts\build.bat          → 编译 EXE
  2. powershell scripts\sign.ps1 → 签名 EXE + 安装证书（一步完成）
  3. cert\deploy.bat            → 部署到 Program Files

  注意：sign.ps1 每次生成新证书，所以每次重新编译后都要重新签名！

【如果已经安装了证书但还是弹窗】
  很可能 EXE 没签名或签名不匹配。验证方法：
  右键 bin\MythwareToolkit.exe → 属性 → 数字签名
  如果这里为空，说明 EXE 没签名，请运行 sign.ps1

【方法一：正确签名（推荐）】
  以管理员 PowerShell 运行：
    powershell -File scripts\sign.ps1
  此脚本自动完成：生成证书 → 安装到受信任根 → 签名 EXE

【方法二：手动安装证书】
  仅当 EXE 已签名但证书未安装时使用。
  管理员 CMD：
    certutil -addstore -f -enterprise Root bin\MythwareToolkit.cer
  管理员 PowerShell：
    Import-Certificate -FilePath bin\MythwareToolkit.cer -CertStoreLocation Cert:\LocalMachine\Root

【方法三：导入原版注册表（兼容旧版）】
  如果使用原版 BengbuGuards 签名的 EXE，双击 cert\RootCA.reg 即可

【方法四：通过 MMC 手动导入】
  1. Win+R → mmc → 确定
  2. 文件 → 添加/删除管理单元 → 证书 → 添加 → 计算机账户 → 本地计算机
  3. 展开"受信任的根证书颁发机构" → 证书
  4. 右键 → 所有任务 → 导入 → 选择 cert\mythware.cer
  5. 一直下一步直到完成

【验证证书是否安装成功】
  certutil -store Root | findstr Mythware
  看到 "MythwareToolkit" 即表示成功

【验证 EXE 是否已签名】
  右键 bin\MythwareToolkit.exe → 属性 → 数字签名 选项卡
  如果为空 → 需要运行 sign.ps1 签名

【终极方案：使用便携版】
  如果以上仍然不行，用便携版免去签名和安装的麻烦：
    scripts\build_portable.bat
  输出 bin\MythwareToolkit_Portable.exe，双击即可运行

【常见错误】
  - "只装证书"：证书和签名是两个东西，必须都做
  - "证书装到当前用户"：必须装到 本地计算机 的 Root，不是当前用户
  - "EXE 不在 Program Files"：UiAccess 程序必须在 C:\Program Files\ 下
  - "编译后没重新签名"：每次编译都会覆盖 EXE，签名丢失
